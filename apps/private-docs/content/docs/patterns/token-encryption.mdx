# Token Encryption Pattern

All OAuth tokens in Huginn are **encrypted at rest** using AES-256-GCM. Encryption/decryption is transparent to service layers — tokens are automatically encrypted when stored and decrypted when retrieved.

## Encryption Implementation

### AES-256-GCM Utilities

```typescript
// From packages/shared/src/services/crypto.ts
import { createCipheriv, createDecipheriv, randomBytes } from "node:crypto";

const ALGORITHM = "aes-256-gcm";
const IV_LENGTH = 12; // GCM standard

function getKey(): Buffer {
  const hex = process.env.CALENDAR_ENCRYPTION_KEY;
  if (!hex || hex.length !== 64) {
    throw new Error(
      "CALENDAR_ENCRYPTION_KEY must be a 64-char hex string (32 bytes). " +
        "Generate one with: node -e \"console.log(require('crypto').randomBytes(32).toString('hex'))\"",
    );
  }
  return Buffer.from(hex, "hex");
}

export function encryptToken(plaintext: string): string {
  const key = getKey();
  const iv = randomBytes(IV_LENGTH);
  const cipher = createCipheriv(ALGORITHM, key, iv);

  const encrypted = Buffer.concat([cipher.update(plaintext, "utf8"), cipher.final()]);
  const tag = cipher.getAuthTag();

  // Format: iv:tag:ciphertext (all base64)
  return [iv.toString("base64"), tag.toString("base64"), encrypted.toString("base64")].join(":");
}

export function decryptToken(encrypted: string): string {
  const key = getKey();
  const parts = encrypted.split(":");
  if (parts.length !== 3) {
    throw new Error("Invalid encrypted token format");
  }

  const iv = Buffer.from(parts[0], "base64");
  const tag = Buffer.from(parts[1], "base64");
  const ciphertext = Buffer.from(parts[2], "base64");

  const decipher = createDecipheriv(ALGORITHM, key, iv);
  decipher.setAuthTag(tag);

  return Buffer.concat([decipher.update(ciphertext), decipher.final()]).toString("utf8");
}
```

## Service Layer Integration

### Transparent Encryption in CalendarConnectionService

```typescript
// From packages/shared/src/services/calendar-connection-service.ts
export function createCalendarConnectionService(db: Database): CalendarConnectionService {
  return {
    async createConnection(data) {
      const [row] = await db
        .insert(calendarConnections)
        .values({
          accountId: data.accountId,
          provider: data.provider,
          providerEmail: data.providerEmail,
          displayName: data.displayName,
          // Tokens automatically encrypted here
          accessToken: encryptToken(data.accessToken),
          refreshToken: encryptToken(data.refreshToken),
          tokenExpiresAt: data.tokenExpiresAt,
        })
        .returning();

      // Return decrypted for immediate use
      return {
        ...toCalendarConnection(row),
        accessToken: data.accessToken, // Return original plaintext
        refreshToken: data.refreshToken,
      };
    },

    async getConnection(id) {
      const row = await db.query.calendarConnections.findFirst({
        where: eq(calendarConnections.id, id),
      });

      if (!row) return null;

      // Automatically decrypt tokens when loading
      return {
        ...toCalendarConnection(row),
        accessToken: decryptToken(row.accessToken),
        refreshToken: decryptToken(row.refreshToken),
      };
    },

    // updateTokens also encrypts automatically...
  };
}
```

**Key Points:**

- **Service layer doesn't know about encryption** — methods work with plaintext
- **Database always stores encrypted tokens**
- **Automatic encryption on write, decryption on read**

## Environment Setup

### Encryption Key Generation

```bash
# Generate 32-byte (256-bit) key as hex string
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
# Output: a1b2c3d4e5f6...64-char-hex-string

# Add to .env
echo "CALENDAR_ENCRYPTION_KEY=a1b2c3d4e5f6..." >> .env
```

### Key Requirements

**Environment Variable:** `CALENDAR_ENCRYPTION_KEY`

- **Length:** Exactly 64 hex characters (32 bytes)
- **Format:** Hex string (0-9, a-f)
- **Generation:** Cryptographically random
- **Rotation:** Not currently supported (future feature)

## Security Properties

### AES-256-GCM Benefits

1. **Authenticated Encryption:** GCM mode provides both confidentiality and authenticity
2. **Random IVs:** Fresh IV per token prevents pattern analysis
3. **Auth Tag:** Detects tampering or corruption
4. **Industry Standard:** NIST-approved, widely audited

### Attack Resistance

**Protection Against:**

- **Data breaches:** Encrypted tokens useless without key
- **Database dumps:** No plaintext tokens in backups
- **Insider threats:** DBAs can't read tokens
- **Token tampering:** GCM auth tag prevents modification

**Does NOT protect against:**

- **Key compromise:** If `CALENDAR_ENCRYPTION_KEY` is leaked, tokens can be decrypted
- **Runtime memory:** Tokens are plaintext in application memory
- **Application compromise:** Running app can decrypt tokens

## Usage in Calendar OAuth Flow

### Token Storage During OAuth

```typescript
// From apps/web/server/api/calendar/callback.ts
export default defineEventHandler(async (event) => {
  const { code, state } = getQuery(event);

  // Verify HMAC-signed state parameter
  const { accountId } = verifyOAuthState(state);

  // Exchange code for tokens
  const tokenResponse = await fetch("https://oauth2.googleapis.com/token", {
    method: "POST",
    body: new URLSearchParams({
      client_id: process.env.GOOGLE_CLIENT_ID!,
      client_secret: process.env.GOOGLE_CLIENT_SECRET!,
      code: code as string,
      grant_type: "authorization_code",
      redirect_uri: `${baseURL}/api/calendar/callback`,
    }),
  });

  const tokens = await tokenResponse.json();
  const userInfo = await fetchGoogleUserInfo(tokens.access_token);

  // Tokens automatically encrypted when stored
  const connectionService = createCalendarConnectionService(db);
  await connectionService.createConnection({
    accountId,
    provider: "google",
    providerEmail: userInfo.email,
    displayName: userInfo.email,
    accessToken: tokens.access_token, // Plaintext input
    refreshToken: tokens.refresh_token, // Plaintext input
    tokenExpiresAt: new Date(Date.now() + tokens.expires_in * 1000),
  });

  return redirect("/calendars", 302);
});
```

### Token Usage in API Calls

```typescript
// From packages/shared/src/services/google-calendar-provider.ts
export const googleCalendarProvider: CalendarProvider = {
  async getEvents(connection, range) {
    // Tokens automatically decrypted when loaded from service
    const response = await fetch(
      `https://www.googleapis.com/calendar/v3/calendars/primary/events?` +
        new URLSearchParams({
          timeMin: range.start.toISOString(),
          timeMax: range.end.toISOString(),
          singleEvents: "true",
          orderBy: "startTime",
        }),
      {
        headers: {
          Authorization: `Bearer ${connection.accessToken}`, // Decrypted token
        },
      },
    );

    // Process response...
  },

  async refreshTokens(connection) {
    // Use refresh token to get new access token
    const response = await fetch("https://oauth2.googleapis.com/token", {
      method: "POST",
      body: new URLSearchParams({
        client_id: process.env.GOOGLE_CLIENT_ID!,
        client_secret: process.env.GOOGLE_CLIENT_SECRET!,
        refresh_token: connection.refreshToken, // Decrypted token
        grant_type: "refresh_token",
      }),
    });

    const tokens = await response.json();

    // Return new tokens (will be encrypted when saved)
    return {
      accessToken: tokens.access_token,
      refreshToken: tokens.refresh_token || connection.refreshToken,
      expiresAt: new Date(Date.now() + tokens.expires_in * 1000),
    };
  },
};
```

## Performance Considerations

### Encryption Overhead

**AES-256-GCM is fast:**

- **Encrypt:** ~1ms per token (64-128 chars)
- **Decrypt:** ~1ms per token
- **Negligible impact** on API response times

### Caching Strategy

**No token caching needed:**

- Tokens loaded infrequently (only when making external API calls)
- Calendar events cached separately (5min TTL)
- Service layer handles decryption transparently

## Testing Strategy

### Unit Tests

```typescript
// Test encryption roundtrip
test("token encryption preserves data", () => {
  const original = "ya29.a0AfH6SMBabc123xyz";
  const encrypted = encryptToken(original);
  const decrypted = decryptToken(encrypted);

  expect(decrypted).toBe(original);
  expect(encrypted).not.toBe(original);
  expect(encrypted.split(":")).toHaveLength(3); // iv:tag:ciphertext
});

// Test invalid format handling
test("decryption rejects invalid format", () => {
  expect(() => decryptToken("invalid")).toThrow("Invalid encrypted token format");
  expect(() => decryptToken("a:b")).toThrow("Invalid encrypted token format");
});
```
