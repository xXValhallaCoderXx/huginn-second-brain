# Note Store v2

The **NoteStore v2** provides comprehensive knowledge management with **semantic search**, **graph linking**, **revision tracking**, and **embedding generation**. Supports **full-text search**, **tag-based filtering**, **ownership isolation**, and **source tracking** for AI agent integration.

## Interface

```typescript
interface NoteStore {
  // Core CRUD
  create(accountId: string, note: CreateNoteInput): Promise<Note>;
  update(noteId: string, accountId: string, updates: UpdateNoteInput): Promise<Note | null>;
  delete(noteId: string, accountId: string): Promise<void>;
  get(noteId: string, accountId: string): Promise<Note | null>;
  list(accountId: string, options?: ListNotesOptions): Promise<Note[]>;
  
  // Search implementations
  search(accountId: string, query: string, limit?: number): Promise<Note[]>;
  searchFullText(accountId: string, query: string, limit?: number): Promise<Note[]>;
  searchSemantic(accountId: string, text: string, limit?: number, minSimilarity?: number): Promise<ScoredNote[]>;
  
  // Graph operations
  link(sourceNoteId: string, targetNoteId: string, relationship: NoteRelationship): Promise<NoteLink>;
  unlink(sourceNoteId: string, targetNoteId: string): Promise<void>;
  getLinks(noteId: string): Promise<NoteLink[]>;
  getGraph(accountId: string): Promise<KnowledgeGraph>;
  getStats(accountId: string): Promise<KnowledgeStats>;
  
  // Embeddings
  embed(text: string): Promise<number[]>;
  upsertEmbedding(noteId: string, content: string): Promise<void>;
  
  // Utility / backfill
  tags(accountId: string): Promise<string[]>;
  listAll(): Promise<Note[]>; // For backfill scripts
}
```

## Core Functionality v2

### 1. Enhanced CRUD Operations

**Create notes with fire-and-forget embeddings:**

```typescript
const note = await noteStore.create(accountId, {
  title: "API Rate Limits",
  content: "OpenRouter has a 1000 req/min limit for free tier",
  tags: ["technical", "api"],
  capturedBy: "user",
  source: { threadId: "tg-chat-123", channel: "telegram" },
});
// Embedding generated asynchronously if OpenRouter key provided
```

**Update with revision tracking:**

```typescript
const updated = await noteStore.update(noteId, accountId, {
  title: "Updated title",
  tags: ["technical", "api", "rate-limits"],
  reason: "Added rate limit details", // Required for revision tracking
});
// Previous state pushed to revisions[] before update
// New embedding generated if content changed
```

### 2. Three Search Implementations

**Backward-compatible ILIKE search:**

```typescript
// Unchanged - for agent tool compatibility
const results = await noteStore.search(accountId, "rate limit", 10);
// Uses: title ILIKE '%rate limit%' OR content ILIKE '%rate limit%'
```

**Full-text search (PostgreSQL tsvector):**

```typescript
const results = await noteStore.searchFullText(accountId, "API rate limiting", 10);
// Uses: search_vector @@ plainto_tsquery('API rate limiting')
// Ordered by: ts_rank(search_vector, query) DESC
```

**Semantic search (pgvector cosine distance):**

```typescript
const results = await noteStore.searchSemantic(accountId, "API rate limiting", 5, 0.7);
// Returns: ScoredNote[] with similarity scores
// Uses: ORDER BY embedding <=> $embedding_vector LIMIT 5
```

### 3. Graph Operations

**Create bidirectional links:**

```typescript
const link = await noteStore.link(sourceNoteId, targetNoteId, "extends");
// Relationship types: "related" | "extends" | "contradicts" | "supersedes"
// Unique constraint prevents duplicate links
```

**Query note relationships:**

```typescript
const links = await noteStore.getLinks(noteId);
// Returns all outbound and inbound links for the note

const graph = await noteStore.getGraph(accountId);
// Returns full knowledge graph: { nodes: GraphNode[], edges: GraphEdge[] }

const stats = await noteStore.getStats(accountId);
// Returns: { noteCount: number, linkCount: number }
```

### 4. Revision Tracking

**Automatic history preservation:**

```typescript
interface NoteRevision {
  title: string;
  content: string;
  reason: string;
  updatedAt: string;
}

// Current note.revisions[] contains all historical states
const current = await noteStore.get(noteId, accountId);
console.log(current.revisions); // Array of all previous versions
```

### 5. Embedding Generation

**Direct embedding API:**

```typescript
const vector = await noteStore.embed("Some text to embed");
// Returns: number[] (1536 dimensions for text-embedding-3-small)

await noteStore.upsertEmbedding(noteId, "Updated content");
// Generates and stores embedding for the note
```

## Key Implementation Details v2

### Ownership & Security (Unchanged)

- **Account isolation:** All queries filtered by `accountId`
- **Ownership checks:** Update/delete operations verify note belongs to account
- **No cross-user access:** Service layer enforces account boundaries
- **Cascade deletes:** Links automatically deleted when notes are deleted

### Search Implementation Details

```sql
-- search() uses ILIKE for fuzzy matching (unchanged)
SELECT * FROM notes
WHERE account_id = $1
  AND (title ILIKE $2 OR content ILIKE $2)
ORDER BY updated_at DESC LIMIT $3;

-- searchFullText() uses PostgreSQL tsvector
SELECT *, ts_rank(search_vector, plainto_tsquery($2)) as rank 
FROM notes
WHERE account_id = $1 
  AND search_vector @@ plainto_tsquery($2)
ORDER BY rank DESC LIMIT $3;

-- searchSemantic() uses pgvector cosine distance
SELECT *, (1 - (embedding <=> $2)) as similarity
FROM notes n
JOIN note_embeddings ne ON n.id = ne.note_id
WHERE n.account_id = $1 
  AND (1 - (embedding <=> $2)) >= $3
ORDER BY embedding <=> $2 LIMIT $4;
```

### Graph Implementation

```sql
-- Links table with bidirectional queries
CREATE TABLE note_links (
  id UUID PRIMARY KEY,
  source_note_id UUID REFERENCES notes(id) ON DELETE CASCADE,
  target_note_id UUID REFERENCES notes(id) ON DELETE CASCADE,
  relationship TEXT CHECK (relationship IN ('related', 'extends', 'contradicts', 'supersedes')),
  created_at TIMESTAMPTZ DEFAULT now(),
  UNIQUE(source_note_id, target_note_id)
);
```

### Embedding Pipeline

**Fire-and-forget pattern:**

```typescript
// In create/update operations
if (this.openrouterApiKey) {
  // Don't await - fire and forget
  this.upsertEmbedding(note.id, `${note.title}\n\n${note.content}`)
    .catch(err => console.error('Embedding generation failed:', err));
}
```

**Rate limiting in backfill:**

```typescript
// apps/agent/src/scripts/backfill-embeddings.ts
for (const note of notes) {
  await noteStore.upsertEmbedding(note.id, `${note.title}\n\n${note.content}`);
  // 200ms rate limiting between requests
  await new Promise(resolve => setTimeout(resolve, 200));
}
```

## Database Schema v2

**Enhanced notes table:**

```sql
CREATE TABLE notes (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  account_id UUID NOT NULL REFERENCES accounts(id) ON DELETE CASCADE,
  title TEXT NOT NULL,
  content TEXT NOT NULL,
  tags TEXT[] NOT NULL DEFAULT '{}',
  source JSONB,
  captured_by TEXT NOT NULL CHECK (captured_by IN ('user', 'agent')),
  revisions JSONB NOT NULL DEFAULT '[]', -- NEW: NoteRevision[]
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  search_vector TSVECTOR GENERATED ALWAYS AS (
    setweight(to_tsvector('english', title), 'A') ||
    setweight(to_tsvector('english', content), 'B')
  ) STORED -- NEW: Full-text search
);

CREATE INDEX notes_search_vector_idx ON notes USING GIN(search_vector);
```

**New embeddings table:**

```sql
CREATE TABLE note_embeddings (
  note_id UUID PRIMARY KEY REFERENCES notes(id) ON DELETE CASCADE,
  embedding VECTOR(1536) NOT NULL, -- OpenAI text-embedding-3-small
  embedded_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE INDEX note_embeddings_vector_idx ON note_embeddings 
USING ivfflat (embedding vector_cosine_ops) WITH (lists = 100);
```

**New links table:**

```sql
CREATE TABLE note_links (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  source_note_id UUID NOT NULL REFERENCES notes(id) ON DELETE CASCADE,
  target_note_id UUID NOT NULL REFERENCES notes(id) ON DELETE CASCADE,
  relationship TEXT NOT NULL CHECK (relationship IN ('related', 'extends', 'contradicts', 'supersedes')),
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  UNIQUE(source_note_id, target_note_id)
);
```
