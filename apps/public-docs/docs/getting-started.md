# Getting started

This starter site gives you a place to publish public-facing docs separately from your internal documentation.

## Local development

From the repository root, run:

```bash
pnpm dev:public-docs
```

The public docs site runs on port `3001` by default so it does not collide with the API app on port `3000`.

## Suggested content structure

A good public docs flow is:

1. Overview
2. Quickstart
3. Concepts
4. Reference
5. Troubleshooting

## Recommended first pages

- installation or signup flow
- your first successful workflow
- key concepts and terminology
- FAQ or troubleshooting guide
